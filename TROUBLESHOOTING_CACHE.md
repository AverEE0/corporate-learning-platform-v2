# Решение проблем с кешированием и отображением изменений

## Если деплой успешен, но изменения не видны:

### 1. Очистка кеша браузера
- **Chrome/Edge**: `Ctrl + Shift + Delete` → выберите "Изображения и файлы в кеше" → "Удалить данные"
- **Firefox**: `Ctrl + Shift + Delete` → выберите "Кеш" → "Удалить сейчас"
- **Или**: `Ctrl + F5` (жесткая перезагрузка страницы)

### 2. Проверка консоли браузера
1. Откройте DevTools (`F12`)
2. Перейдите на вкладку "Console"
3. Проверьте наличие ошибок (красные сообщения)
4. Если есть ошибки импорта или компонентов - сообщите разработчику

### 3. Проверка Network вкладки
1. Откройте DevTools (`F12`)
2. Перейдите на вкладку "Network"
3. Обновите страницу (`F5`)
4. Проверьте, что файлы `.js` загружаются с новыми именами (Next.js генерирует новые имена при каждом билде)

### 4. Проверка на сервере
Если у вас есть SSH доступ:
```bash
ssh root@your-server
cd /root/corporate-learning-platform-v2
git log --oneline -3  # Проверьте последние коммиты
docker compose logs app | tail -50  # Проверьте логи приложения
docker compose restart app  # Перезапустите приложение
```

### 5. Принудительная пересборка
Если изменения не применяются, возможно нужно пересобрать Docker образ:
```bash
# На сервере
cd /root/corporate-learning-platform-v2
docker compose down
docker compose build --no-cache app
docker compose up -d
```

### 6. Проверка версии Next.js билда
В консоли браузера выполните:
```javascript
// Проверьте версию билда
console.log(window.__NEXT_DATA__?.buildId)
```

### 7. Очистка кеша Next.js на сервере
```bash
# На сервере
cd /root/corporate-learning-platform-v2
rm -rf .next
docker compose restart app
```

## Частые проблемы:

### Проблема: Боковая панель не отображается
**Решение:**
1. Проверьте консоль браузера на ошибки импорта
2. Убедитесь, что файл `components/course-sidebar.tsx` существует
3. Проверьте, что импорт правильный: `import { CourseSidebar } from "@/components/course-sidebar"`

### Проблема: Галочки не появляются
**Решение:**
1. Убедитесь, что вы ответили на вопросы (для блоков)
2. Проверьте, что прогресс урока достиг 100% (для уроков)
3. Обновите страницу после ответа на вопрос

### Проблема: Анимации не работают
**Решение:**
1. Проверьте, что `framer-motion` установлен: `npm list framer-motion`
2. Проверьте консоль браузера на ошибки JavaScript
3. Убедитесь, что JavaScript включен в браузере

## Если ничего не помогает:

1. **Полная очистка кеша:**
   - Закройте все вкладки с сайтом
   - Очистите кеш браузера полностью
   - Откройте сайт в режиме инкогнито (`Ctrl + Shift + N`)

2. **Проверка GitHub Actions:**
   - Откройте: https://github.com/AverEE0/corporate-learning-platform-v2/actions
   - Убедитесь, что последний деплой завершился успешно (зеленая галочка)
   - Проверьте логи деплоя на наличие ошибок

3. **Свяжитесь с разработчиком:**
   - Сообщите, какие именно изменения не видны
   - Приложите скриншот консоли браузера (если есть ошибки)
   - Укажите версию браузера и ОС
